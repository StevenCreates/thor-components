# Welcome Thor Components

# Getting Started

This component library was built to help build out UI appications quick and easy but with flexibility to style however you need.

## Dependencies

`npm install --save-dev @babel/core @babel/preset-env @babel/preset-react babel-loader webpack`

**Component-dependent Dependencies**

`npm install --save-dev @babel/plugin-proposal-nullish-coalescing-operator @babel/plugin-proposal-optional-chaining`

## babel.config.js

Two presets (`@babel/env` and `@babel/preset-react`) will need to be added to the Babel config:

```javascript
module.exports = {
   ...
   presets: [
      ...
      '@babel/env',
      '@babel/preset-react',
   ],
}
```

Depending on if it's necessary for the component, a few plugins may need to be added as well:

```javascript
module.exports = {
   ...
   plugins: [
      ...
      '@babel/plugin-proposal-nullish-coalescing-operator',
      '@babel/plugin-proposal-optional-chaining',
   ],
}
```

## webpack.config.js

Webpack will need to be configured with the appropriate loaders for the thor-components components:

```javascript
module.exports = {
   ...,
   module: {
      rules: [
         ...,
         {
            test: /\.js$/,
            include: path.resolve(__dirname, 'node_modules/thor-components/'),
            exclude: [/__tests__/],
            loader: 'babel-loader',
            query: {
               presets: ['@babel/preset-react'],
            },
         },
      ],
   },
}
```

If using `npm link` to test thor-components components in another app locally, the following will need to be added:

```javascript
module.exports = {
   ...,
   resolve: {
      symlinks: false,
   },
}
```

## jest.config.js

A transform pattern will also need to be added to the jest config:

```javascript
module.exports = {
   ...,
   transformIgnorePatterns: ['/node_modules/(?!(thor-components)/).*/'],
}
```
